三個運算子的功能：

運算子意思什麼時候用
-連續範圍「3880 到 4380 之間的票都可以」
;並列 (OR)「這幾個價位都可以，沒有先後之分，由『排序優先』下拉決定」
>優先順序「先搶左邊；左邊完全沒票才退到右邊」

實際寫法範例：

寫法意思
（留空）不限價位，所有可用區域都列入候選
4480只接受 4480 元的區域
4,480同上（千分位逗號自動忽略）
4480-66804480 到 6680 之間（連續區間）
6680-4480同 4480-6680，順序無關，自動 swap 成 4480-6680
-50005000 以下（無下限）
4480-4480 以上（無上限）
4480;66804480 或 6680 都接受（並列，由「排序優先」決定先後）
4480>6680👉 先搶 4480；如果 4480 完全沒票，才退到 6680
6680>4480👉 先搶 6680；沒票才退到 4480
4480-5500>3880先搶 4480~5500 範圍內的票；整個範圍都沒票才退到 3880
4480>3880;5500先搶 4480；沒票才退到「3880 或 5500」（後者由排序優先決定）
3000-4500;6680;8000-9000三段並列，由排序優先決定

運算子優先級規則：; 比 > 結合得更緊（像數學的 × 比 + 緊）。 所以 A>B;C 讀作「先 A；沒票才試 B 或 C」，不是「先 (A 或 B)；沒票才試 C」。

選 ; 還是 >？

  • 想同時保留多個選項（例如「我可以接受 4480 或 6680，看哪個剩比較多就抓」） → 用 ;
  • 想嚴格按優先順序（例如「一定要 6680，沒了才將就 4480」） → 用 >

票價數字必須 ≥ 100 才會被當價錢解析（避免把「剩餘 5」誤判成價錢 5 元）。

通過所有篩選後，要先抓哪一個。同票價時改用剩餘張數當第二排序鍵。
方向 第 名
範例：方向「從多到少」+ 第 1 名 = 抓剩最多的（預設）；第 2 名 = 抓第二多 — 用來閃開「大家都搶最多票區」的擁擠時段；方向「從少到多」+ 第 1 名 = 搶救快賣完的稀有區。候選不夠多時自動 fallback 到最後一個可用候選。

② 是否售完該區域 <font> 不能是「已售完」跳過此區域

常見問題：

  • 所有區域都被濾掉 → 跟其他模式一樣，依上方「區域自動遞補」開關決定：關 = 不停 reload 頁面等票，開 = 放寬條件從全部可用區挑。
  • 智能模式下，「區域關鍵字」欄位中的空白會被當作「無意義字元」忽略（不再是 AND 分隔）。如果要表達 OR，請用 ;。
  • 場館頁面如果不是把票價寫在每個區域行內（例如某些活動只在區段標題寫「3980-7880」、子區行只有「(best available) 已售完」）→ 該活動的「接受的價位」可能無法生效，請改用「區域關鍵字」鎖定區段。

限制：目前只有 TixCraft 一般售票流程支援（右側區域列表那種）。SVG 純圖型座位圖、Ticketmaster JSON zone 流程不適用，會 fallback 到「從上到下」行為。

🧠 智能模式生效　此欄位現在使用「智能匹配」規則，跟其他模式不一樣：
  • 紅1區 → 自動匹配紅1A/B/C/D/E 整組（區後綴可選、英文字母可加可不加）
  • 紅 1 區、紅　1　區 → 全/半形空白都會被忽略，等同 紅1區
  • 紅1區;紫1區 → 多個區域用 ; 表示 OR
  • 留空 = 不過濾位置，純按「接受的價位」+「排序優先」決定
※ 智能模式下空白不再代表 AND；想 AND 直接在關鍵字內接著寫即可（如 紅1A）。
