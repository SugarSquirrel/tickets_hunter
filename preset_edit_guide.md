# 多設定檔切換 / 編輯 完整指南

> 在搶票進行中安全地編輯其他帳號的設定，不影響當前 bot

## 三個操作模式對照表

| 模式 | 動到 settings.json？ | 影響搶票中的 bot？ | 何時用 |
|---|---|---|---|
| **載入** | 會（整個覆蓋）+ 頁面 reload | 🚨 **會** | 搶完換帳號、bot 沒在跑時 |
| **📝 編輯** | 不會 | ✅ 不會 | **搶票中**編輯其他 preset |
| **另存為…** | 會 | 🚨 **會** | 把目前畫面內容存成新 preset |
| **存到 preset**（編輯模式按鈕） | 不會 | ✅ 不會 | 編輯後存回該 preset 檔 |

## 場景一：搶票前準備多份設定

```
1. 啟動 bot 前
2. 在 UI 填好帳號 A 的所有欄位
3. 按「存檔」（寫入 settings.json）
4. 按「另存為…」→ 輸入「帳號A」
5. 修改欄位變成帳號 B 的設定
6. 按「存檔」
7. 按「另存為…」→ 輸入「帳號B」
8. 重複到所有帳號都有 preset
```

完成後 `src/presets/` 會有：`帳號A.json`、`帳號B.json`、`帳號C.json`…

## 場景二：搶票中編輯其他帳號的 preset

**重點：絕對不要按「載入」或「另存為…」或底部的「存檔」按鈕**，那些都會覆蓋 settings.json 影響搶票中的 bot。

```
A 帳搶票中（bot 用 settings.json = 帳號A 內容跑）
            │
            ▼
1. 下拉選單選「帳號B」
2. 按「📝 編輯」← 黃色按鈕
3. 表單變成帳號B的內容（但 settings.json 沒動）
4. 頂部跳出黃色 banner：
   ┌─────────────────────────────────────────┐
   │ 🔧 編輯中：帳號B（不影響搶票中的設定）   │
   │ 💾 存到 preset    ✖ 結束編輯           │
   └─────────────────────────────────────────┘
5. 修改你要改的欄位（網址、Cookie、票數、區域關鍵字、價位…）
6. 按 banner 上的「💾 存到 preset」
   → 寫進 src/presets/帳號B.json
   → settings.json 完全沒動
   → 搶票中的 bot 繼續用帳號A 跑，零影響
7. 想再編別的：下拉選「帳號C」→ 再按「📝 編輯」
   （會問你「未存的修改會遺失，繼續嗎？」如果你還沒按存到 preset）
8. 編完所有 preset 後，按「✖ 結束編輯」
   → 表單重新載入 settings.json（變回顯示帳號A 內容）
```

## 場景三：A 帳搶完 → 切到 B 帳繼續搶

```
1. 帳號A 搶到票，手動完成結帳付款
2. 確認瀏覽器頁面已到付款完成頁
3. 按底部「結束」按鈕（會把 bot subprocess 也關掉）
4. 重新打開設定頁面（或如果還開著就直接用）
5. 下拉選單選「帳號B」
6. 按「載入」← 藍色按鈕
   → settings.json 被覆蓋成帳號B 內容
   → 頁面自動 reload
7. 確認所有欄位是帳號B 的內容
8. 按「搶票」啟動新的 bot
```

## 安全機制（已內建）

進入「📝 編輯」模式後：

| 按鈕 | 狀態 | 為什麼 |
|---|---|---|
| 載入 / 另存為 / 編輯 / 刪除 | **全部 disable** | 防誤觸覆蓋 settings.json |
| 底部「存檔」 | **變紅色** + 文字變「⚠️ 存檔（會覆蓋 settings.json，影響搶票中）」 | 還可以按但會明確警告 |
| 下拉選單 | 可用 | 讓你看其他 preset 名稱 |
| Banner 上的 💾 存到 preset | 可用 | 編輯模式專用的安全儲存 |
| Banner 上的 ✖ 結束編輯 | 可用 | 離開編輯模式時用 |

## 常見 Q&A

**Q: 為什麼編輯模式下「另存為」是 disabled 的？**
A: 另存為內部會先 save 表單到 settings.json 再 copy，這會干擾搶票中的 bot。如果你想存成新名字，先「✖ 結束編輯」回到正常模式，再用「另存為」。

**Q: 我已經編輯模式中，但忘了「💾 存到 preset」就按了「✖ 結束編輯」會怎樣？**
A: 你的修改會被丟掉（表單會被 settings.json 蓋回去）。preset 檔本身不會壞。

**Q: 兩個 bot 同時跑（多帳號），可以同時編輯第三個 preset 嗎？**
A: 可以。設定 GUI 只有一份，編輯第三個 preset 不會動 settings.json，兩個 bot 都不受影響。但**注意**：如果你之後按了「載入」切到第三個 preset，兩個 bot 都會被 reload 影響。

**Q: 怎麼快速知道現在 bot 在用哪份設定？**
A: 看 terminal log 第一行的 `homepage`、`tixcraft_sid` 等欄位印出來的內容，對照各 preset 檔。也可以在 file explorer 開 `src/presets/帳號A.json` 跟 `src/settings.json` 比對內容。

**Q: 改 preset 是不是要 bot 重啟才生效？**
A: 編輯 mode 寫入 preset 檔，**檔案改了但 settings.json 沒改**，所以 bot 不會自動切換。要切換得：(1) 結束目前 bot → (2)「載入」該 preset → (3) 重新搶票。或者：搶完後再切。

**Q: preset 檔放哪？會上 GitHub 嗎？**
A: `src/presets/<名稱>.json`。`.gitignore` 已經把 `src/presets/` 整個排除，**不會上 GitHub**。

## 配合「動作速度倍率」的進階用法

每個 preset 可以存自己的 `action_speed_multiplier` 值：

- 帳號A（主帳，謹慎）: 1.0
- 帳號B（備帳，激進）: 0.5
- 帳號C（衝最快）: 0.3

切換時 multiplier 也跟著切換，不用每次手動改。

---

最後重要提醒：**結束按鈕現在會把所有追蹤到的 bot subprocess 一起 terminate**（之前的修正）。所以你要切換帳號時用「結束」是安全的，不會留下殭屍 bot 在背景偷偷搶票。
